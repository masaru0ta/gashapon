# 仕様書ルール
1.以下の項目を持つこと
  - 基本情報（概要・対応デバイス）
  - 画面設計（モック資料リンク・画面項目定義）
  - 処理ロジック
  - データ設計（データ元、データ構造、API設計）
  - 非機能要件（パフォーマンス要件）
  - テスト方針

2.概要は分かりやすい文章で、誰が何のために使う機能かを書くこと

3.画面が必要な機能ではモックアップ資料が必ずリンクされていること

4.画面項目定義はモックアップ資料にある画面項目と一致していること

5.処理ロジックにはエッジケースの動作が記載されていること

6.データ設計にデータの取得元が書かれていること

7.テスト方針には細かなテスト項目ではなく、どういうテスト項目の作り方をするか記載すること
